How the dataset is built

Methodology

This launch repo is a curated GitHub Pages directory, not a full event platform or official publication.

1. Source policy

Prefer official organizer pages, ticketing pages, Meetup event pages, Eventbrite listings, university event pages, and established community portals. Social/community posts are discovery aids, not equal-trust replacements.

2. Inclusion rule

Include events happening in Australia during 2026 that are clearly related to AI, ML, data science, generative AI, AI engineering, analytics, or closely adjacent technical communities.

3. Location focus

Melbourne and Sydney get extra weight because they consistently surface higher AI/data event density, while other capitals are included when reliable public metadata is visible.

4. Dedup logic

Potential duplicates should be collapsed using normalized title, date, city, and organizer, preferring the most official canonical URL.

5. Confidence

High confidence means visible title/date/city/source. Medium and low confidence are retained only when the event is plausibly relevant but the topic fit is less direct.

6. Expansion plan

The repo structure is intentionally built to scale toward 100–200 verified events as more official listings, university calendars, and local tech communities are curated.

Last-mile curation is manual by design so the data stays human-readable and Git-friendly.
